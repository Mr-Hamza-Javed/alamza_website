SETUP GUIDE

Website Setup Guide

Is guide mein sab kuch hai: text aur photos kaise badalni hain, Firebase rules kaise lagani hain, aur admin panel kaise use karna hai.

1

Folder mein kya kya hai?

Alamza Industry.dc.html Main website — customers yeh dekhte hain
Admin Panel.dc.html Messages aur intern submissions dekhne ke liye (editable source)
Alamza Admin.html Admin panel — ab website ke saath hi public folder mein rehta hai; website ke footer “ADMIN” link se bhi khulta hai. (Login zaroori.)
site-data.json Website ka SARA TEXT — text badalna ho to yehi file edit karein (Step 2)
firebase-config.js Firebase keys — lag chuki hain ✓
site-content.js Technical helpers — isse haath na lagayen
assets/ (folder) Aap ki saari images isi folder mein (logo, photos) — Step 3 dekhein
alamza-admin-core.js Admin ka helper (Admin Panel.dc.html ke saath) — haath na lagayen
GUIDE (admin tab) Yeh guide — ab Admin Panel ke andar hai (login ke baad GUIDE tab). Alag file ki zaroorat nahi.
2

Website ka text kaise badlein?

Sara text site-data.json file mein hai — company name, headline, products, team ke naam, phone number, sab kuch. Yeh static file hai isi liye website itni tez khulti hai. Products, team, gallery aur showcase ab lists (arrays) hain — aap jitni chahein utni entries add ya remove kar sakte hain.

SITE-DATA.JSON SE YEH SAB BADAL SAKTE HAIN
• Products — "products" → "items" ek list hai. Har item mein title, code, desc, img. Jitne items list mein honge, utne cards website par aayenge.
• Showcase — "showcase" → "items". Har item mein title, code, desc, img, ek specs list, aur gradientOpacity (image par gradient ki gehrai — neeche dekhein).
• Team — "team" → "members" list. Har member: name, role, img, alt. 4, 5, 6 ya jitne chahein.
• Gallery — "gallery" → "images" list. Har image: img, alt. List khali ho to gallery section website par dikhta hi nahi (neeche dekhein).
• Social links — "social" mein Facebook, Instagram, YouTube, TikTok, LinkedIn ke link. Khali "" wala icon footer mein nahi aayega. WhatsApp "contact" → "whatsapp" se.
• Contact — phone, WhatsApp number, email aur address.
LIST MEIN ENTRY ADD / REMOVE KAISE KAREIN?
Ek list [ ... ] ke andar hoti hai, aur har entry { ... } hoti hai jisko comma , alag karta hai. Naya member add karne ke liye poora { ... } copy karke comma ke baad chipka dein:
"members": [
  { "name": "Ali Raza", "role": "CNC Operator", "img": "team-1.jpg", "alt": "Ali Raza" },
  { "name": "Sana Khan", "role": "QC Engineer",  "img": "team-2.jpg", "alt": "Sana Khan" }
]
Remove karne ke liye us entry ka { ... } (aur uske saath wala comma) hata dein. Aakhri entry ke baad comma nahi hota — yeh dhyan rahe.
SHOWCASE — HAR IMAGE PAR GRADIENT KAM/ZIADA
Har showcase item mein "gradientOpacity" hai — 0 (bilkul saaf) se 1 (sab se gehra) tak. Agar koi image pehle hi dark hai to value kam rakhein (jaise 0.25); agar bright hai to thodi ziada (jaise 0.6). Default 0.45 hai. Image ke corners khud panel mein blend ho jaate hain.
a) site-data.json ko kisi bhi editor mein kholen (Notepad bhi chalega).
b) Sirf "quotes ke andar" ka text badlein. Misal: "title": "FORGED IN SAHIWAL." → "title": "BUILT IN SAHIWAL."
c) File save karein — website refresh karne par naya text aa jayega. Hosting par hai to file dobara upload kar dein.
Khayal rahe: Quotes (") aur commas (,) ko mat torein — sirf unke andar ka text badlein. Agar website khali ho jaye to aakhri change wapis kar dein.

4

Intern form ko control karna

Intern form ki har cheez site-data.json → "intern" se control hoti hai — koi code change nahi.

FILE UPLOAD AUR LINKS — ON / OFF
• "uploadEnabled": true = file upload box dikhega (ab optional hai, zaroori nahi). false = upload band.
• "urlsEnabled": true = student apne kaam ke links (URLs) add kar sakta hai — ek se ziada bhi (“+ ADD ANOTHER LINK”). false = links band.
• Dono false ho to form mein na upload aayega na links — sirf baaki fields rahenge.
FIELDS — ZAROORI / OPTIONAL, AUR ANDAR KA TEXT
"intern" → "fields" mein har field ke 3 control hain:
"name": { "label": "FULL NAME", "placeholder": "e.g. Ahmad Raza", "required": true }
• "required" — true rakhein to field zaroori (*), false rakhein to (OPTIONAL).
• "placeholder" — box ke andar dikhne wala halka text (jaise “e.g. Ahmad Raza”). Jo yahan likhenge wohi box mein aayega.
• "label" — field ke upar ka naam.
DEPARTMENT LIST
"intern" → "departments" ek simple list hai. Naya department add karne ke liye quotes mein naam likh kar comma ke saath add karein, remove karne ke liye hata dein:
"departments": ["Production", "CNC Machining", "Quality Control", "Casting / Foundry"]
5

Firebase — security (sab se zaroori)

Yeh step aapka data mehfooz banata hai. Maqsad: koi bhi banda form bhar (SUBMIT) sake, lekin koi bhi data PARH ya DELETE na kar sake — sirf aap (logged-in admin). Firebase console (console.firebase.google.com → alamza-web) mein yeh 4 cheezein karni hain.

Pehle samajh lein: Firebase ki keys (firebase-config.js) public hona normal aur safe hai — har website mein aisi hi hoti hain. Asli hifazat Rules + Authentication se aati hai (jo server par lagti hai), file chhupane se nahi.
CONSOLE MEIN YEH 4 KAAM
a) Build → Realtime Database enable karein (agar pehle se nahi).
b) Build → Storage enable karein (intern files ke liye).
c) Build → Authentication → Get started → Sign-in method mein Email/Password ko Enable karein. Phir Users → Add user se apna admin email + password banayein. Yehi se admin panel khulega.
d) Neeche di gayi Rules Database aur Storage dono mein paste karke Publish karein.
REALTIME DATABASE → RULES (copy/paste)
{
  "rules": {
    "contactMessages": {
      ".read": "auth != null",
      "$id": {
        ".write": "auth != null || (!data.exists() && newData.exists())",
        ".validate": "newData.hasChildren(['name','message'])"
      }
    },
    "internSubmissions": {
      ".read": "auth != null",
      "$id": {
        ".write": "auth != null || (!data.exists() && newData.exists())",
        ".validate": "newData.hasChildren(['name'])"
      }
    },
    "meta": {
      "storageUsed": {
        ".read": true,
        ".write": true,
        ".validate": "newData.isNumber()"
      }
    }
  }
}
STORAGE → RULES (copy/paste)
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /intern-uploads/{fileName} {
      allow read, delete: if request.auth != null;
      allow create: if request.resource.size < 30 * 1024 * 1024;
    }
  }
}
YEH RULES KYA KARTI HAIN?
• SUBMIT (write): koi bhi visitor contact form aur intern form bhar sakta hai — nayi entry ban jaati hai. ✅
• READ / DELETE: koi visitor messages ya submissions na parh sakta hai, na badal/delete kar sakta hai. Sirf logged-in admin (auth != null). 🔒
• Files: visitor sirf 30 MB tak file upload kar sakta hai; parhna/delete sirf admin. Files ka download link bhi sirf admin ke andar banta hai.
• meta/storageUsed sirf ek number hai (storage gauge) — koi private data nahi.
Khayal rahe: Authentication mein Email/Password ke ilawa koi aur sign-in method enable na karein, aur naya "Add user" sirf aap karein. Iske bina koi admin nahi ban sakta. Apna password kisi ko na batayein.
6

Admin Panel

• Admin ek hi standalone file hai — Alamza Admin.html — jo ab website ke saath hi public folder mein rehti hai.
• Isse kholne ke 2 tarike: (1) website ke footer mein “ADMIN” link, ya (2) seedha your-site.com/Alamza Admin.html.
• Login email + password se hota hai — wohi jo aapne Firebase console (Step 5c) mein banaya. Koi default password nahi. Login ke baghair andar ka kuch nahi khulta.
• Password bhool gaye? Login screen par usi naam ka link hai — ya seedha Firebase console → alamza-web → Authentication → Users mein apne user ka password reset kar dein.
• File ke top par ALAMZA_WEBSITE_URL mein apni live website ka link daalein (“OPEN WEBSITE” button ke liye).
• Settings tab se apna password kabhi bhi badal sakte hain. Messages: contact messages + WhatsApp reply. Intern Submissions: details, files (download/delete) aur diye gaye links.

Deploy karna — kaun si file kahan

Aasaan usool: poora folder public host par upload kar dein — sirf yeh guide chhod kar. Admin bhi public folder mein rehta hai; uska taala login hai, na ke file chhupana.

PUBLIC — YEH SAB UPLOAD KAREIN (ek hi folder)
Alamza Industry.dc.html  ·  Alamza Admin.html  ·  Admin Panel.dc.html  ·  alamza-admin-core.js  ·  support.js  ·  site-content.js  ·  firebase-config.js  ·  site-data.json  ·  assets/ (saari images)
Poora folder kisi bhi static host par daal dein (Firebase Hosting / Netlify / GitHub Pages / cPanel).
GUIDE — AB ADMIN KE ANDAR (login ke baad)
Yeh guide ab kisi alag file mein nahi — seedha Admin Panel ke GUIDE tab mein hai, jo sirf login ke baad khulta hai. Is mein koi password/secret nahi, isliye koi security issue nahi. Sab kuch public deploy kar sakte hain.
Security ki asli baat: Admin public folder mein hone ke bawajood mehfooz hai, kyunke uska taala Firebase Email/Password login + Rules hai (Step 5). Chahe koi admin file khol bhi le, aapke login ke baghair woh koi data nahi parh ya delete kar sakta — sirf form submit ho sakta hai.
