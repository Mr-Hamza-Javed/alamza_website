2

Website ka text kaise badlein?

Sara text site-data.json file mein hai — company name, headline, products, team ke naam, phone number, sab kuch. Yeh static file hai isi liye website itni tez khulti hai. Products, team, gallery aur showcase ab lists (arrays) hain — aap jitni chahein utni entries add ya remove kar sakte hain.

SITE-DATA.JSON SE YEH SAB BADAL SAKTE HAIN
• Products — "products" → "items" ek list hai. Har item mein title, code, desc, img. Jitne items list mein honge, utne cards website par aayenge.
• Showcase — "showcase" → "items". Har item mein title, code, desc, img, ek specs list, aur gradientOpacity (image par gradient ki gehrai \u2014 neeche dekhein).
• Team — "team" → "members" list. Har member: name, role, img, alt. 4, 5, 6 ya jitne chahein.
• Gallery — "gallery" → "images" list. Har image: img, alt. List khali ho to gallery section website par dikhta hi nahi (neeche dekhein).
• Social links — "social" mein Facebook, Instagram, YouTube, TikTok, LinkedIn ke link. Khali "" wala icon footer mein nahi aayega. WhatsApp "contact" → "whatsapp" se.
• Contact — phone, WhatsApp number, email aur address.
LIST MEIN ENTRY ADD / REMOVE KAISE KAREIN?
Ek list [ ... ] ke andar hoti hai, aur har entry { ... } hoti hai jisko comma , alag karta hai. Naya member add karne ke liye poora { ... } copy karke comma ke baad chipka dein:
"members": [
  { "name": "Ali Raza", "role": "CNC Operator", "img": "team-1.jpg", "alt": "Ali Raza" },
  { "name": "Sana Khan", "role": "QC Engineer",  "img": "team-2.jpg", "alt": "Sana Khan" }
]
Remove karne ke liye us entry ka { ... } (aur uske saath wala comma) hata dein. Aakhri entry ke baad comma nahi hota \u2014 yeh dhyan rahe.
SHOWCASE \u2014 HAR IMAGE PAR GRADIENT KAM/ZIADA
Har showcase item mein "gradientOpacity" hai \u2014 0 (bilkul saaf) se 1 (sab se gehra) tak. Agar koi image pehle hi dark hai to value kam rakhein (jaise 0.25); agar bright hai to thodi ziada (jaise 0.6). Default 0.45 hai. Image ke corners khud panel mein blend ho jaate hain.
a) site-data.json ko kisi bhi editor mein kholen (Notepad bhi chalega).
b) Sirf "quotes ke andar" ka text badlein. Misal: "title": "FORGED IN SAHIWAL." → "title": "BUILT IN SAHIWAL."
c) File save karein — website refresh karne par naya text aa jayega. Hosting par hai to file dobara upload kar dein.
Khayal rahe: Quotes (") aur commas (,) ko mat torein — sirf unke andar ka text badlein. Agar website khali ho jaye to aakhri change wapis kar dein.

4

Intern form ko control karna

Intern form ki har cheez site-data.json → "intern" se control hoti hai \u2014 koi code change nahi.

FILE UPLOAD AUR LINKS \u2014 ON / OFF
• "uploadEnabled": true = file upload box dikhega (ab optional hai, zaroori nahi). false = upload band.
• "urlsEnabled": true = student apne kaam ke links (URLs) add kar sakta hai \u2014 ek se ziada bhi (\u201C+ ADD ANOTHER LINK\u201D). false = links band.
• Dono false ho to form mein na upload aayega na links \u2014 sirf baaki fields rahenge.
FIELDS \u2014 ZAROORI / OPTIONAL, AUR ANDAR KA TEXT
"intern" → "fields" mein har field ke 3 control hain:
"name": { "label": "FULL NAME", "placeholder": "e.g. Ahmad Raza", "required": true }
• "required" \u2014 true rakhein to field zaroori (*), false rakhein to (OPTIONAL).
• "placeholder" \u2014 box ke andar dikhne wala halka text (jaise \u201Ce.g. Ahmad Raza\u201D). Jo yahan likhenge wohi box mein aayega.
• "label" \u2014 field ke upar ka naam.
DEPARTMENT LIST
"intern" → "departments" ek simple list hai. Naya department add karne ke liye quotes mein naam likh kar comma ke saath add karein, remove karne ke liye hata dein:
"departments": ["Production", "CNC Machining", "Quality Control", "Casting / Foundry"]
5

Firebase — rules lagana

Aap ki keys already lag chuki hain ✓. Bas yeh check kar lein ke Firebase console (console.firebase.google.com → alamza-web) mein Realtime Database aur Storage dono enable hain, aur unki Rules yeh hain:

REALTIME DATABASE → RULES (copy/paste)
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
STORAGE → RULES (copy/paste)
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /intern-uploads/{fileName} {
      allow read, write: if true;
    }
  }
}
Storage ka usool: Firebase Storage sirf intern files ke liye use hota hai (upar wali rule bhi sirf intern-uploads/ folder kholti hai). Messages, submissions ka data aur admin password — sab Realtime Database mein jata hai.
Khayal rahe: Yeh rules sab ko read/write ki ijazat deti hain — simple password system ke liye yehi zaroori hai, lekin security basic hai. Sensitive cheezein database mein na rakhein, aur admin password kisi ko na batayen.
6

Admin Panel

• Admin website se bilkul alag hai — Alamza Admin.html ek hi standalone file hai jise aap kisi alag (private) jagah host karte hain. Website ke folder mein rakhne ki zaroorat nahi.
• File ke top par ALAMZA_WEBSITE_URL mein apni live website ka link daal dein (“OPEN WEBSITE” button ke liye).
• Default password: alamza2026 — pehli dafa login karke Settings tab se zaroor badal lein.
• Messages tab: contact form ke messages realtime — WhatsApp reply ka button bhi.
• Intern Submissions tab: har intern ki poori details, uski files (download/delete) aur jo links usne diye — sab dikhte hain. Limits: 30 MB per file, total 100 MB.
7

Speed aur hosting

• Website ka text, photos aur design — sab static files hain, isliye pehla paint bohat tez hai. Database ka koi intezar nahi.
• Firebase ka code page khulne ke baad background mein load hota hai — sirf forms ke liye chahiye hota hai.
• Host karne ke liye: poora folder jaisa hai waisa kisi bhi static host par upload kar dein (Firebase Hosting, Netlify, ya cPanel). Saari files aik hi folder mein rehni chahiyen.
